# raw.vives — Plan y guion de vídeo

## Objetivo

Presentar en 60–75 segundos la identidad, el recorrido y la profundidad técnica sin convertir la pieza en una demo acelerada. Formato maestro 16:9 a 1440p, con adaptación vertical posterior.

## Orden recomendado

| Tiempo | Plano | Acción | Texto opcional |
| --- | --- | --- | --- |
| 00–05 | Negro / intro | Marca e inicialización | `raw.vives / visual archive` |
| 05–12 | Hero | Entrada y scroll lento | `Stories between light and silence` |
| 12–22 | Narrativa | Manifiesto, historia y capítulos | `A personal visual archive` |
| 22–34 | Archivo | Búsqueda, categoría y orden | `30 photographs / 2022—2025` |
| 34–46 | Detalle | Abrir imagen, anterior/siguiente | `Every image has context` |
| 46–52 | Fullscreen | Abrir y cerrar con Escape | `Built for focus` |
| 52–59 | Interacción | Cursor y WebGL, captura real | `Progressive motion` |
| 59–67 | Mobile | Hero, menú y filtros | `Bilingual and responsive` |
| 67–75 | Cierre | Marca y dominio | `gallery.aleviclop.dev` |

## Guion de voz

“raw.vives nace para convertir un archivo personal en una experiencia editorial. Cada fotografía forma parte de una secuencia, pero conserva su propia historia. La navegación permite buscar, filtrar y compartir sin perder contexto. Motion, cursor y WebGL aportan atmósfera de forma progresiva; el contenido sigue funcionando con teclado, movimiento reducido y dispositivos más modestos. El resultado une fotografía, diseño y desarrollo en un archivo bilingüe preparado para crecer.”

## Captura

- Grabar el build final mediante `pnpm preview`, no el servidor de desarrollo.
- Ocultar marcadores, extensiones, cursor del sistema innecesario y datos personales.
- Mantener scroll estable; evitar zoom digital y aceleraciones que oculten la interfaz.
- Capturar desktop a 1440×900 y móvil a 390×844.
- Usar una sesión limpia para registrar la intro una sola vez.
- Mostrar el cursor contextual y WebGL solo si quedan legibles en la grabación.
- No representar un comparador: el producto actual no incluye esa función.
- Terminar con el dominio solo después de validar que sirve la release publicada.

## Edición y sonido

Ritmo pausado, cortes por acción, tipografía mínima y música con licencia verificable. Mantener sonido por debajo de la voz, exportar H.264 de alta calidad y conservar un máster sin textos para adaptaciones.

## Estado

Se entrega plan, guion y capturas estáticas. No se genera un vídeo ficticio ni se publica material externo en esta fase.
